## Engineering Philosophy

I care less about trendy stacks and more about understanding systems deeply.

Things I think about while building:

* What happens when traffic spikes?
* What breaks under concurrency?
* How difficult is this to debug?
* Where are the architectural bottlenecks?
* What happens when requirements change?
* Can another engineer maintain this 6 months later?

Clean code matters.
But resilient systems matter more.
